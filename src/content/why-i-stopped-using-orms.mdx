After three years of reaching for Prisma and Drizzle by default, I have been writing raw SQL more often. Here is what changed my mind.

## The Abstraction Leak

Every ORM promises to let you think in objects instead of tables. What they deliver is a leaky abstraction that forces you to think in both — simultaneously, and usually at 2 AM when a query is slow.

The specific moment that changed my mind: I spent four hours debugging a Prisma query that was generating a nested SELECT for every row in a 10,000-row table. The fix in raw SQL was a single JOIN. The fix in Prisma was "do not do that."

## What I Actually Need

I do not need type-safe query builders. I need:

- **Type-safe query results** — knowing what columns come back
- **Migration tooling** — versioned, reversible schema changes
- **Connection pooling** — handled correctly

Raw SQL with a good type system (TypeScript + generated types from your schema) gives you all three without the overhead.

## The Exception

I still use ORMs for prototypes and internal tools where query performance does not matter. The productivity boost is real when you are building a CRUD admin panel at 11 PM.

But for anything that ships to users? I want to see the SQL.
